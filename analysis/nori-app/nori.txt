- Nori > App related features
    - app-integrity
        - Flows
            - integrity-check > Check folders and files are in place and semantically correct, self heal
            - authentication-check > Check user is authenticated properly to use repos and Claude code, self heal
            - autoupdate > Check for new updates, update system automatically, self heal
        - Triggers
            - onAppStart > integrity-check and authentication-check and autoUpdate
    - app-settings
        - Flows
            - switch-ui-mode > Switch the ui between light mode and dark mode
        - Triggers
            - uiModeButton > onClick > switch-ui-mode
- Vault
    - Vault settings
        - flows
            - vault-registration > User can define one or more vaults (git repos)
            - vault-link-project > User can define which registered vaults as used in which projects
        - triggers
            - vaultCreateButton > onClick > vault-registration
            - vaultLinkProject > onClick > vault-link-project
    - Vault Life cycle
        - Flows
            - vault-pull > pull changes from the git
            - vault-push > Push changes to git
            - regenerate-db > Based on md files, create a table with all indexed data
            - vector-embedding-system > Create vector system
            - vault-reconciliation > Compare files and advice user about conflicts if exist
            - audit-vault > validates all frontmappers and file context and the db creation
        - Triggers
            - OnAppStart > vault-reconciliation and generate-db and vecto-embedding-system
            - vaultSyncButton > onClick - vault pull
            - auditButton > audit-vault
    - Knowledge Life cycle
        - Flows
            - create-knowledge > Creates a new md file with the frontmapper specifications and content based on use input. It uses claude code with clear instructions on how to create and validate the knowledge. audit-knowledge runs after creation
            - edit-knowledge > Allow user to modify the knowledge (content and frontmapper). audit-knowledge runs after creation
            - delete-knowledge > Allow user to remove a knowledge
            - audit-knowledge > Validates the knowledge is something AI would not do and frontmapper is well setup
        - Triggers
            - createknowledgeButton > create-knowledge and audit-knowledge and regenerate-db
            - editKnowledgeButton > edit-knowledge and audit-knowledge and regenerate-db
            - deleteKnolwedgeButton > delete-knolwedge and regenerate-db
            - auditKowledgeButton > audit-knowledge
